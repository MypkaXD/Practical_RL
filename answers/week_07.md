# Применение RNN для генерации последовательностей

Применение RNN для генерации последовательностей использует архитектуру Encoder-Decoder, где энкодер считывает входные данные и формирует скрытое состояние, а декодер пошагово генерирует выходную последовательность. Ключевая проблема заключается в различии между режимами обучения и инференса: при обучении декодер получает эталонные токены (teacher forcing), а при генерации подаёт на вход свои собственные предсказания. Это приводит к distribution shift — распределение генерируемых последовательностей отличается от обучающего, ошибки накапливаются, и модель может начать выдавать «мусорные» результаты. Именно эти ограничения supervised-подхода мотивируют переход к reinforcement learning для прямой оптимизации качества генерации.

# Применение RL к дообучению моделей генерации последовательностей

Проблема supervised learning - при обычном обучении с учителем модель тренируется на эталонных парах (референсах). Это требует большого размеченного датасета и приводит к проблеме distribution shift — на этапе генерации модель сталкивается с распределением данных, отличным от обучающего, что ведёт к накоплению ошибок.

RL решает проблему distribution shift, потому что модель обучается на своих собственных сгенерированных последовательностях, а не на фиксированных референсах. Для этого нужна только функция награды $R(x, y)$, которая оценивает качество сгенерированного ответа $y$ на вход $x$. При этом энкодер остаётся замороженным, а доучивается только декодер.

Для уменьшения шума при оценке градиента используется подход Actor-Critic, где роль критика выполняет сама модель. В качестве базовой линии (baseline) берётся награда, полученная при жадном инференсе (greedy decoding) — то есть при детерминированном выборе токена с максимальной вероятностью.

Формула градиента принимает следующий вид:

$\nabla J = E[ \nabla \log \pi_\theta(a | s) *A(s, a) ]$

где преимущество $A(s,a)$ вычисляется как разница между наградой за текущее действие и наградой за baseline-действие:

$$A(s, a) = R(s, a) - R(s, a_{\text{inference}}(s))$$

RL позволяет дообучить модель генерации последовательностей, оптимизируя непосредственно целевую метрику, а не максимизируя правдоподобие эталонных данных. Это решает проблему distribution shift и снижает зависимость от размера размеченного датасета.
