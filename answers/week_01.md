# Базовые понятия Reinforcement Learning

RL (Обучение с подкреплением) - метод поиска оптимальной стратегии методом проб и ошибок, с помощью взаимодействия с окружающей средой при получении rewards (награды) за свои действия. 

Цель - максимизировать rewards.

Среда (environment) - внешний мир, в которой работает агент. 
Агент (agent) - "игрок", который принимает различные решения и совершает действия
Действие (action) - какой-либо выбор агента в состоянии_k, например, на основе ojbservation
Наблюдение (observation) - информация от среды
Состояние (state) - вся информация о среде (агенту может быть доступна только часть этой информации)
Награда (reward) - сигнал от среды за какое-то действие агента

$s$ - состояние среды из общего множества состояний $S$
$s_0$ - начальное состояние среды
$r$ - reward
$(s_0, a_0, r_0, ..., s_k, a_k, r_k)$ - сессия (последовательность состояние, действие, награда)
$a$ - действие агента из общего множества действий $A$
$\sum_i {r_i}=R$ - суммарная награда за сессию
Цель обучения - найти политику, при которой $E[R] \to max$   
Политика $\pi(a|s)$ - какой действие выполнить при состоянии $s$ 

# Cross Entropy Method (CEM):

CEM - итеративный алгоритм для задач RL, основанный на отборе элитных сессия (elite sessions). 

Делаем k-сессий, выбираем лучшие (elite sessions), обновляем на их основе политику, повторяем. 

## Табличный

Подходит для задач, когда состояние среды является дискретным (непрерывные состояния среды можно также свести к дискретным) + этих состояний не должно быть колоссальное число. Действия также должны быть дискретными

Составляется матрица размера count_of_states $*$ count_of_actions (вероятностная матрица), нормированная по строкам

$\pi(a|s)=A_{a,s}$ 

Делаем k-sessions, выбираем elites sessions (допустим какой-то перцентиль от награды)
Стоит отметить, что при совершении k-сессий, важно сохранить историю сессии + суммарную награду

После чего обновляем политику на основе elite-sessions:

$$\pi_{new}​(a∣s)=\frac{CountElite(s,a)}{CountElite(s)}​$$
числитель - кол-во выбора действия a для состояния s
знаменатель - кол-во раз, когда были в состоянии s

## Нейронный

Подходит, когда состояние среды непрерывно или их число колоссальное. Тогда для аппроксимации таблицы используют neural networks (NN). Цель NN - вернуть вероятность действия a при состоянии s (вернуть вектор вероятностей для всех действий)

1. Init weights of NN random
2. Play k-sessions
3. Select elites
4. Update weights of NN $W_{i+1}=W_{i}+\alpha*\nabla{\sum_{s_{j}, a_{j} \in elites}{log(\pi(a_j|s_j))}}$      

NN пытается научиться максимизировать правдоподобие действий из elite sessions:
$\pi = argmax_{\pi} \sum_{s_i, a_i \in Elite} \log \pi (a_i|s_i)$ 

В случае, если действия также являются непрерывным пространством, то используют нормальное распределение:
$\pi_{W}(a|s)=N(\mu(s), \sigma^2)$
